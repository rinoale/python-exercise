---
name: Type blocks must match explanations
description: Lesson Type: blocks must directly demonstrate the claims made in the explanation text, not unrelated toy examples
type: feedback
---

Type: blocks in lessons must be directly connected to the explanation above them — they should prove or demonstrate the specific claim being made.

**Why:** User got frustrated multiple times because Type: blocks were disconnected toy examples that showed shapes/API calls but didn't help understand the concept. E.g., Step 6 explained "position makes order matter" but the Type: block just showed shapes.

**How to apply:** Before writing a Type: block, identify the specific claim the step makes, then write code that proves that claim. Also: never define a class/function in the lesson code without showing it in the lesson text — hidden definitions are useless to the learner.
